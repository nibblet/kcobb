# Cobb Brain Lab: Memoir → Stories → Decision DNA → Principles (v2)

Scaffold and implement the pipeline under `**cobb_brain_lab/**` with an auditable manifest spine, defensive TOC handling, a real validator, normalization before clustering, better clustering outputs, and doctrine as the final artifact.

---

## 1. Directory layout

```
cobb_brain_lab/
  book.pdf                    # User-provided
  README.md
  requirements.txt
  Makefile                    # make split | validate | cluster | doctrine
  .gitignore                  # book.pdf, out/pdf/, out/txt/; optionally keep out/extracted/
  prompts/
    extraction_prompt.md
  scripts/
    00_inspect_pdf.py         # Diagnose PDF / TOC before splitting
    01_split_by_toc.py        # Split + write manifest
    02_extract_units.py        # Pydantic validator + manifest coverage
    03_cluster_principles.py  # Normalize → dedup → cluster → draft doctrine
  out/
    manifest.jsonl             # Spine: story_id, title, pages, paths, word/char count, optional hash
    pdf/
    txt/
    extracted/
    clusters/
  doctrine/                   # Final artifact
    core_principles.md
    heuristics.md
    voice_style.md
    system_prompt_v1.md
```

Use `**.gitkeep**` in empty output dirs; **paths everywhere cwd-independent** via `BASE = Path(__file__).resolve().parent.parent` in scripts.

## 2. Manifest (spine for the pipeline)

**2.1** `**out/manifest.jsonl`** (or `**out/manifest.csv`**) — Generated by **01_split_by_toc.py** only. One record per story.

Fields: `story_id`, `title`, `printed_page_start` / `printed_page_end`, `pdf_page_start` / `pdf_page_end`, `output_pdf_path`, `output_txt_path`, `word_count`, `char_count`, `hash` (optional).

**2.2** Downstream: **02_extract_units.py** reads manifest for coverage (missing JSON, orphan JSON, coverage %). **03** can use manifest for supporting stories in doctrine. Everything references this spine.

---

## 3. scripts/02_extract_units.py (stub)

Blueprint: extraction is either (A) human-in-loop in Cursor (paste story + prompt, save JSON to `out/extracted/`) or (B) future API. So **02** will not implement an API call**. Plan:

- **Option A (recommended):** Minimal script that (1) ensures `out/extracted/` exists, (2) optionally validates all `*.json` in that dir against the extraction schema (e.g. required keys: `story_title`, `principles`, `decision_heuristics`, etc.) and reports missing/invalid files. No API keys, no provider-specific code.
- **Option B:** Pure stub: print a short message pointing to `prompts/extraction_prompt.md` and `out/extracted/`, and exit.

Either way, the pipeline stays “run 01 → drop JSON into out/extracted (by hand or future script) → run 03.”

---

## 4. scripts/00_inspect_pdf.py (helper)

Read-only. Prints: **total pages**; **first 5 pages** (first 3 lines each); **which pages contain "Contents"**; **which page contains the anchor title** (first non-skipped TOC entry). Diagnose in ~20 seconds.

## 6. What you’ll do after scaffold

1. Add your memoir PDF as `book.pdf`.
2. If the TOC format differs, adjust `TOC_PAGES` and `TOC_LINE_RE` in `01_split_by_toc.py`.
3. Run 01 → get `out/txt/###_Story_Title.txt`.
4. Extract 5–10 stories into `out/extracted/*.json` (Cursor + prompt or future 02).
5. Run 03 → inspect `out/clusters/cluster_summary.md` and distill to a core doctrine set.
